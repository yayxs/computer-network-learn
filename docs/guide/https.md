## `HTTPS`是如何进行加密的 谈谈`https`的原理？为什么 https 能保证安全？

> HTTPS 并不是应用层的一种新协议 **部分的通信接口** 使用`SSL` 和 `TLS` 协议代替

**加密处理和认证** 添加了加密以及认证机制的 http 称为 https 。SSL 采用公开秘钥加密

### 非对称加密

一把私有秘钥 一把公开密钥

### CA 机构

- 提出公开秘钥的申请
- 分配给一个公开的秘钥
- 服务器送给客户端
- 客户端拿到数字证书（证书）

## `HTTPS`安全通信

- HTTP+加密处理+认证+完整性保护==>HTTPS
- 通信时的协议：https://
